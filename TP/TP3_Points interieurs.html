Mthode de point intrieur

L'objectif de ce TP est d'utiliser des mthodes de point intrieur pour minimiser une fonction sous contrainte. $newcommand{\R}{\mathbb{R}}$ $newcommand{\umin}[1]{\underset{#1}{\min}\;}$ $DeclareMathOperator{\eqdef}{\overset{\tiny def}{=}}$

Soit $f$ une fonction rgulire de $\mathbb{R}^n$ dans $\mathbb{R}$, nous nous intresserons deux types de problme :

  1. Le problme contrainte linaire $$ (\mathcal{P}_{\infty,\text{ lin.}}) \qquad \umin{x\in \mathbb{R}^n, A x \leq b} f(x) $$ o $A$ est une matrice $\mathcal M_{pn}(\mathbb{R})$ et $b\in \mathbb{R}^p$.
  2. Le problme avec une contrainte non linaire $$ (\mathcal{P}_{\infty,\text{ non lin.}}) \qquad \umin{x\in \mathbb{R}^n, g(x)\le 0} f(x) $$ O $g$ est une fonction rgulire de $\mathbb{R}^n$ dans $\mathbb{R}$.

L'ide des mthodes de point intrieur est d'approximer les problmes $(\mathcal P_{\infty,\bullet})$ en utilisant des fonctions barrires logarithmiques

$$ (\mathcal{P}_{t, \text{non-lin.}}) \qquad \umin{x\in \mathbb{R}^d, } f_t(x) \eqdef f(x) - \frac{1}{t} \text{ln}( -g(x) ) $$ ou $$ (\mathcal{P}_{t, \text{ lin.}}) \qquad \umin{x\in \mathbb{R}^d, } f_t(x) \eqdef f(x) - \frac{1}{t} \text{Log}( b-A x ) $$ avec $$ \text{Log}(u) \eqdef \sum_i \ln(u_i) $$

Ainsi la fonction -Log ou -log qui est une fonction strictement concave, agit comme une barrire pour la contrainte. On s'attend ce qu' la limite, quand $t$ tend vers $+\infty$, le problme $(\mathcal P_{t,\bullet})$ tende vers le problme $(\mathcal P_{\infty,\bullet})$.

On suppose dans cette section que l'on se place dans le cadre d'une seule contrainte non-linaire. Si $f$ et $g$ sont donnes et $f_t=f - \frac{1}{t} \text{log}( -g)$,

Calculez ci-dessous le gradient et la Hessienne de $f_t$ en fonction des gradients et de la Hessienne de $f$ et de $g$ $$\nabla f_t(x) = ?? \quad H [f_t](x) = ?? $$

$$ f_t(x) = f(x) - \frac{1}{t}\ln(-g(x)).$$

Gradient

$$ \nabla f_t(x) = \nabla f(x) - \frac{1}{t}\,\frac{1}{g(x)}\,\nabla g(x). $$

Hessienne

$$ H[f_t](x) = H[f](x) - \frac{1}{t}\left( \frac{1}{g(x)}\,H[g](x) - \frac{1}{g(x)^2}\,\nabla g(x)\nabla g(x)^{\top} \right). $$

class non_lin_cst() :

        self.zeros()
        self.f=f
        self.g=g
        self.t=t
        self.nb_constraints=nb_constraints # nombre de contraintes du problme
    def zeros(self) :
        self.nb_eval=0 # number of evaluations of the function self.value()
        self.nb_grad=0 # number of evaluations of the function self.grad()
        self.nb_hess=0 # number of evaluations of the function self.Hess()
    def value(self,x) :
        # returns the value of the function at point x
        self.nb_eval += 1
        l = np.log(-self.g.value(x)) if self.g.value(x) < 0 else -np.inf
        return self.f.value(x) - 1/self.t * l
    def grad(self,x) :
        # returns the gradient of the function at point x
        self.nb_grad += 1
        return self.f.grad(x) - 1/(self.t*self.g.value(x)) * self.g.grad(x)

        #Il faut reshape le gradient sinon a ne fonctionne pas

        le_grad = self.g.grad(x).reshape((2,1))

def ls_wolfe(x, function, step, descent, f, df):
    """
    Recherche linaire satisfaisant les conditions de Wolfe.
    
    Paramtres
    ----------
    x : ndarray
        Point courant
    function : objet
        Doit fournir value(x) et grad(x)
    step : float
        Pas initial
    descent : ndarray
        Direction de descente
    f : float
        Valeur f(x)
    df : ndarray
        Gradient f(x)
    """

    # Bornes sur le pas
    step_max = np.inf
    step_min = 0

    # Constantes de Wolfe
    eps1 = 1e-4   # condition d'Armijo (dcroissance suffisante)
    eps2 = 0.9    # condition de courbure

    # Initialisation
    step2 = step
    x2 = x + step2 * descent
    f2 = function.value(x2)
    df2 = function.grad(x2)

    # Boucle jusqu' satisfaction des conditions de Wolfe
    while (f2 > f + eps1 * step2 * np.dot(df, descent)) \
          or (np.dot(df2, descent) < eps2 * np.dot(df, descent)):

        # Condition d'Armijo non satisfaite : pas trop grand
        if f2 > f + eps1 * step2 * np.dot(df, descent):
            step_max = step2
            step2 = (step_min + step_max) / 2

        # Condition de courbure non satisfaite : pas trop petit
        else:
            step_min = step2
            if step_max == np.inf:
                step2 = 2 * step2
            else:
                step2 = (step_min + step_max) / 2

        # Mise  jour du point test
        x2 = x + step2 * descent
        f2 = function.value(x2)
        df2 = function.grad(x2)

    # Retour du point acceptable et du pas associ
    return x2, f2, df2, step2


def ls_wolfe_step_is_one(x, function, step, descent, f, df):
    """
    Version simplifie : on force le pas initial  1
    (classique en mthode de Newton/quasi-Newton)
    """
    return ls_wolfe(x, function, 1., descent, f, df)


def dc_Newton(x, function, df):
    """
    Calcul d'une direction de descente de type Newton,
    avec repli sur la descente de gradient si ncessaire.
    """

    # Hessienne au point courant
    Hess = function.Hess(x)

    # Direction de Newton
    descent = -np.linalg.inv(Hess) @ df

    # Cosinus de l'angle entre la direction et -f
    costheta = (np.dot(descent, -df)) / (
        np.linalg.norm(descent, 2) * np.linalg.norm(df, 2)
    )

    # Si la direction de Newton n'est pas suffisamment descendante,
    # on utilise la descente de gradient
    if costheta <= 0.1:
        descent = -df

    return descent

La mthode du chemin central

Notre objectif est d'envoyer le paramtre $t$ vers $+\infty$. Pour ce faire on va suivre la mthode du chemin central, nous commenons par nous donner $t_1$ et nous notons $x_0$ le point initial et $x_1$ le minimiseur donn de $f_{t_1}$ par une mthode de Newton commenant $x_0$. Ensuite nous multiplions $t_1$ par $\mu$ pour obtenir $t_2$ et nous lanons une mthode de Newton commenant $x_1$. On note $x_2$ le rsultat obtenu. Ainsi nous allons construire une suite de points solutions du problme $$x_k \in \textrm{argmin} f_{t_k}(x).$$ avec $t_{k}=\mu t_{k-1}$ et $x_{k}$ une solution donne par un algorithme de Newton (avec recherche de Wolfe) et avec comme point de dpart $x_{k-1}$. L'ide d'utiliser comme initialisation la solution d'un problme d'optimisation s'appelle "warm restart".

On peut montrer que l'erreur que l'on fait entre $x_k$ et $x^\star$ (o $x^\star$ est le minimum de $f$) vrifie : $$ 0 \le f(x_k)-f(x^\star)\le \frac{p}{t_k},$$ o $p$ est le nombre de contraintes. Ainsi on va s'arrter ds que $\varepsilon t_k > p$ o $\varepsilon$ est la prcision de l'algorithme de Newton. Ecrire une fonction def central_path(function,mu,varepsilon,x0) qui implmente cette stratgie. Elle prend en argument

  1. function: une instance de la classe non_lin_cst
  2. mu: un rel $>1$ qui est le facteur multiplicatif de t
  3. varepsilon: une prcision
  4. x0: un point de dpart de la mthode

Cette fonction doit rendre costs,x o

  1. costs est la liste des cots pour toutes les itrations
  2. x est le point final d'arrive

On rappelle que la valeur de $p$ se trouve dans function.nb_constraints tandis que la valeur de $t$ se trouve dans function.t

def central_path(function, mu, varepsilon, x0):
    """
    Mthode du chemin central pour un problme  barrire.

    Paramtres
    ----------
    function : objet
        Fonction barrire f_t, avec attribut t et nb_constraints
    mu : float
        Facteur de mise  jour du paramtre de barrire (mu > 1)
    varepsilon : float
        Prcision demande
    x0 : ndarray
        Point initial (strictement admissible)

    Retour
    ------
    costs : list
        Historique des cots totaux
    xk : ndarray
        Approximation finale du minimiseur
    """

    # Paramtre de barrire initial
    tk = function.t

    # Liste des cots cumuls
    costs = []

    # Initialisation du point courant
    xk = x0

    # Critre d'arrt thorique des mthodes de point intrieur
    # (varepsilon * t >= m, o m = nombre de contraintes)
    while varepsilon * tk < function.nb_constraints:

        # Minimisation (approche) de f_t via Newton + Wolfe
        res = opt.main_algorithm(
            function,
            10,                 # nombre max d'itrations de Newton
            xk,
            dc_Newton,          # direction de Newton scurise
            ls_wolfe_step_is_one,  # recherche linaire de Wolfe
            tol=varepsilon,
            verbose=False
        )

        # Dernier point obtenu (minimiseur de f_t)
        xk = res['list_x'][-1]

        # Cot cumul des itrations de Newton
        newton_cost = np.sum(res['list_costs'])

        # Valeur de la fonction barrire au point final
        c_cost = function.value(xk)

        # Cot total (Newton + barrire)
        total_cost = newton_cost + c_cost
        costs.append(total_cost)

        # Mise  jour du paramtre de barrire
        tk = mu * tk
        function.t = tk

    return costs, xk

class lin_cst() :
    def __init__(self,f,A,b,t, nb_constraints=1.) :
        self.zeros()
        self.f=f
        self.A=A
        self.b=b
        self.t=t
        self.nb_constraints=nb_constraints # nombre de contraintes du problme
    def zeros(self) :
        self.nb_eval=0 # number of evaluations of the function self.value()
        self.nb_grad=0 # number of evaluations of the function self.grad()
        self.nb_hess=0 # number of evaluations of the function self.Hess()
    def value(self,x) :
        # returns the value of the function at point x
        self.nb_eval += 1
        if min(self.b - self.A@x) <= 0:
            return np.inf
        return self.f.value(x) - 1/self.t * np.sum(np.log(self.b - self.A@x))
    def grad(self,x) :
        # returns the gradient of the function at point x
        self.nb_grad += 1
        grad = self.f.grad(x) + 1/self.t * np.sum(A/(self.b-self.A@x).reshape((np.shape(A)[0],1)),axis = 0)
        return grad.reshape(np.shape(x))#somme colonnes de a divis par les coefs de a pour cahque ligne
    def Hess(self,x) :
        # returns the Hessian of the function at point x
        #Il faut reshape le gradient sinon a ne fonctionne pas
        self.nb_hess += 1
        hess = self.f.Hess(x) + 1/self.t * np.transpose(A)@np.diag(1/(self.b-self.A@x)**2)@A
        return hess

Afin de re-crire le problme du Lasso comme un problme d'optimisation lisse sans contrainte, on introduit, pour tout vecteur $w\in \mathbb{R}^p$, on introduit le vecteur $x=(x_-,x_+)$ tel que $x_-=\max(-w,0)$ et $x_+=\max(w,0)$. On a ainsi toujours $$w = x_+ - x_-\quad \text{et}\quad |w|=x_++x_-\quad \text{et}\quad x\ge 0.$$ Le problme revient donc minimiser, sous la contrainte $x\ge 0$ et en dcomposant $x=(x_+,x_-)$ $$ f(x) = \frac{1}{2}\Vert B(x_+-x_-)-y\Vert_2^2 + \lambda \langle x,1\rangle, $$ o $1$ est le vecteur rempli de $1$. Ainsi la contrainte s'crit bien $Ax \le b$ avec $A=-\text{Id}_{2p}$, $b=0$.

class lasso() :
    def __init__(self,B,lam,y) :
        self.B = B
        self.lam = lam
        self.y = y
        self.p = B.shape[1]
        self.nb_eval = 0 # number of evaluations of the function self.value()
        self.nb_grad = 0 # number of evaluations of the function self.grad()
        self.nb_hess = 0 # number of evaluations of the function self.Hess
        

    def zeros(self):
        ()

    def value(self, x):
        self.nb_eval += 1
        p = self.p

        x_minus = x[:p]
        x_plus  = x[p:]

        w = x_plus - x_minus
        r = self.B @ w - self.y

        return 0.5 * np.dot(r, r) + self.lam * np.sum(x)

    def grad(self,x) :
        self.nb_grad += 1
        p = self.p

        x_minus = x[:p]
        x_plus  = x[p:]

        w = x_plus - x_minus
        r = self.B @ w - self.y
        Br = self.B.T @ r

        g_minus = -Br + self.lam * np.ones(p)
        g_plus  =  Br + self.lam * np.ones(p)

        return np.concatenate([g_minus, g_plus])
         


    def Hess(self,x) :
        self.nb_hess += 1
        BtB = B.T @ B
        return np.block([
            [ BtB,    -BtB ],
            [-BtB,     BtB ]
        ])
